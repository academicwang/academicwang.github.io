时间序列的四个成分
约 45 分钟

什么是时间序列？

时间序列（Time Series）就是按时间顺序排列的一系列数据点。简单来说，就是把同一个指标在不同时间点的观测值记录下来，按时间先后排成一串。

比如：

  • 某商店每月的销售额
  • 某城市每天的平均气温
  • 某公司每季度的利润

时间序列的核心思想是：过去的数据中隐藏着某种规律，我们可以把这种规律提取出来，用于预测未来。

四个成分的直观理解

任何一个时间序列，都可以想象成由四个"力量"共同作用产生的：

趋势 (Trend)
长期的、缓慢的总体方向。比如一家成长型企业的销售额逐年稳步上升，这就是趋势。
季节变动 (Seasonal)
固定周期的规律性波动。比如羽绒服销量每年冬天达到高峰，夏天跌到谷底。
周期变动 (Cyclical)
非固定周期的中长期波动。比如经济繁荣—衰退—复苏的循环，周期不固定。
随机变动 (Random)
无法预测的偶然波动。比如某天突降暴雨导致客流骤减，属于随机扰动。

加法模型与乘法模型

如何把这四个成分组合起来？有两种经典模型：

加法模型 当四个成分相互独立、以绝对数值相加时：
$$Y = T + S + C + R$$

其中：$Y$ = 观测值，$T$ = 趋势，$S$ = 季节变动，$C$ = 周期变动，$R$ = 随机变动。

乘法模型 当四个成分相互影响、以比例相乘时：
$$Y = T \times S \times C \times R$$

乘法模型更适合季节波动的幅度随趋势增大而增大的场景（如销售额越高，季节性波动绝对值也越大）。

什么时候用加法，什么时候用乘法？

  • 如果季节波动的绝对幅度基本不变 → 用加法模型
  • 如果季节波动的相对比例基本不变（即波动幅度随趋势增长） → 用乘法模型

商业数据分析中，销售额、利润等财务数据通常使用乘法模型；气温、库存量等数据常使用加法模型。

本节小结

  • 时间序列 = 按时间顺序排列的数据点
  • 四个成分：Trend（趋势）、Seasonal（季节）、Cyclical（周期）、Random（随机）
  • 加法模型：$Y = T + S + C + R$（波动幅度固定）
  • 乘法模型：$Y = T \times S \times C \times R$（波动比例固定）
  • 销售/利润数据常用乘法模型；气温/库存常用加法模型

移动平均法与趋势计算
约 60 分钟 · 含完整公式推导

为什么需要移动平均？

我们想从时间序列中提取趋势成分 $T$，但原始数据里混杂着季节波动、周期波动和随机波动。移动平均的核心思想是：用"取平均"的方式把这些短期波动"抹平"。

想象一下：你记录每天的气温，某些天突然降温只是偶然天气现象。如果把连续几天的气温取平均，这些偶然波动就会被削弱，呈现出更稳定的"基础气温"。

简单移动平均（Simple Moving Average）

最简单的移动平均是取最近 $n$ 个数据点的算术平均值。

定义 设时间序列的观测值为 $Y_1, Y_2, ..., Y_t, ...$
公式 $n$ 期简单移动平均（SMA）为：
$$M_t = \frac{Y_t + Y_{t-1} + Y_{t-2} + ... + Y_{t-n+1}}{n} = \frac{1}{n} \sum_{i=0}^{n-1} Y_{t-i}$$
3期移动平均示例
$$M_3 = \frac{Y_3 + Y_2 + Y_1}{3}$$ $$M_4 = \frac{Y_4 + Y_3 + Y_2}{3}$$ $$M_5 = \frac{Y_5 + Y_4 + Y_3}{3}$$

注意一个关键问题：如果数据有季度性（比如4个季度为一个周期），用3期移动平均就不太合适——你会混合不同季节的数据，结果依然会有季节残留。

黄金法则：移动平均的期数应等于季节周期的长度。对于季度数据（周期=4），应该用4期移动平均；对于月度数据（周期=12），应该用12期移动平均。

中心化移动平均（Centered Moving Average）

当季节周期为偶数（如季度数据周期=4）时，直接用4期移动平均会产生一个尴尬的问题：计算出的平均值落在两个季度的"中间"，而不是某个具体季度上。

问题 对季度数据（Q1, Q2, Q3, Q4）取4期移动平均：
$$M_{2.5} = \frac{Y_1 + Y_2 + Y_3 + Y_4}{4}$$

这个值对应的是第2期和第3期之间的"中点"，而不是Q1、Q2、Q3或Q4本身！

解决方案 计算两个连续4期移动平均的平均值，这就是中心化移动平均（CMA）。
步骤1 先计算两个连续的4期移动平均：
$$A_1 = \frac{Y_1 + Y_2 + Y_3 + Y_4}{4}$$ $$A_2 = \frac{Y_2 + Y_3 + Y_4 + Y_5}{4}$$
步骤2 取这两个移动平均的平均值，得到中心化移动平均，它正好落在第3期（Q3）上：
$$CMA_3 = \frac{A_1 + A_2}{2} = \frac{1}{2}\left(\frac{Y_1+Y_2+Y_3+Y_4}{4} + \frac{Y_2+Y_3+Y_4+Y_5}{4}\right)$$
简化 合并同类项：
$$CMA_3 = \frac{Y_1 + 2Y_2 + 2Y_3 + 2Y_4 + Y_5}{8}$$

这等价于一个"加权"的5点平均，首尾权重为1，中间权重为2。

为什么移动平均能"消除"季节波动？

这是移动平均最核心的数学原理，我们来逐步推导：

假设 使用加法模型：$Y_t = T_t + S_t + R_t$（暂时忽略周期变动C）
第1步 对4个季度取移动平均（假设周期为4）：
$$M = \frac{Y_1 + Y_2 + Y_3 + Y_4}{4}$$
第2步 将每个 $Y_t$ 分解为趋势+季节+随机：
$$M = \frac{(T_1+S_1+R_1) + (T_2+S_2+R_2) + (T_3+S_3+R_3) + (T_4+S_4+R_4)}{4}$$
第3步 重新分组：
$$M = \frac{T_1+T_2+T_3+T_4}{4} + \frac{S_1+S_2+S_3+S_4}{4} + \frac{R_1+R_2+R_3+R_4}{4}$$
第4步 关键观察——季节指数在一个完整周期内之和为0（加法模型）：
$$S_1 + S_2 + S_3 + S_4 = 0$$

因此季节成分的平均值完全抵消了！

第5步 随机成分的平均值也会趋近于0（正负抵消）：
$$\frac{R_1+R_2+R_3+R_4}{4} \approx 0$$
结论
$$M \approx \frac{T_1+T_2+T_3+T_4}{4} \approx T_{中心点}$$

移动平均后，只剩下趋势成分！这就是移动平均能提取趋势的原理。

例题：用移动平均提取趋势

例题：某企业2021-2023年季度销售额（万元）
时间Q1Q2Q3Q4
2021806070120
2022906575135
20231007285150

求2021年Q3的中心化移动平均（即趋势估计值）

解：

步骤1：计算第一个4期移动平均（对应Q2-Q3之间）：

$$A_1 = \frac{80 + 60 + 70 + 120}{4} = \frac{330}{4} = 82.5$$

步骤2：计算第二个4期移动平均（对应Q3-Q4之间）：

$$A_2 = \frac{60 + 70 + 120 + 90}{4} = \frac{340}{4} = 85.0$$

步骤3：中心化移动平均（对应Q3）：

$$CMA_{2021Q3} = \frac{82.5 + 85.0}{2} = \frac{167.5}{2} = 83.75$$

结论：2021年Q3的趋势估计值为 83.75 万元。

对比实际值70万元，差异 $70 - 83.75 = -13.75$ 万元，这就是该季度的季节变动（Q3通常是淡季，低于趋势）。

Python实现：移动平均计算

# Python: 计算中心化移动平均
import pandas as pd
import numpy as np

# 季度销售额数据（2021 Q1 到 2023 Q4，共12个季度）
data = [80, 60, 70, 120,   # 2021年
        90, 65, 75, 135,   # 2022年
        100, 72, 85, 150]  # 2023年

dates = pd.date_range('2021-01-01', periods=12, freq='Q')
series = pd.Series(data, index=dates)

# 第1步：计算4期移动平均（未中心化）
ma4 = series.rolling(window=4).mean()

# 第2步：中心化移动平均 = 两个连续4期MA的平均
cma = ma4.rolling(window=2).mean()

print("原始数据:", series.values)
print("4期移动平均:", ma4.round(2).values)
print("中心化移动平均:", cma.round(2).values)

本节小结

  • 简单移动平均：$M_t = \frac{1}{n}\sum_{i=0}^{n-1} Y_{t-i}$
  • 移动平均的期数应等于季节周期长度
  • 偶数周期需要做中心化处理：两个连续MA再取平均
  • 数学原理：季节指数在一个完整周期内之和为0，因此取平均后被抵消
  • 移动平均的结果 ≈ 趋势成分

季节变动预测与Python实践
约 60 分钟 · 完整预测流程

提取季节指数

当我们用移动平均得到趋势值 $T$ 后，就可以从原始数据中"剥离"趋势，得到季节成分。

加法模型 季节变动 = 观测值 − 趋势值
$$S = Y - T$$
乘法模型 季节指数 = 观测值 ÷ 趋势值
$$S = \frac{Y}{T}$$

乘法模型中，季节指数通常在1附近波动：大于1表示"旺季"，小于1表示"淡季"。

计算平均季节变动

同一季度在不同年份会有不同的季节指数（因为随机因素的影响）。我们需要把各年份同一季度的季节指数取平均，得到该季度的"典型"季节指数。

步骤 对每个季度，收集所有年份的季节指数，计算算术平均值：

加法模型（季节变动）：

$$\bar{S}_{Q1} = \frac{S_{2021,Q1} + S_{2022,Q1} + S_{2023,Q1}}{\text{年数}}$$

乘法模型（季节指数）：

$$\bar{S}_{Q1} = \frac{S_{2021,Q1} + S_{2022,Q1} + S_{2023,Q1}}{\text{年数}}$$

预测公式：趋势外推 + 季节调整

有了趋势和季节指数，我们就可以预测未来了。预测的基本思路是：

  1. 用线性回归等方法把趋势外推到未来
  2. 把季节指数叠加到外推趋势上
步骤1：趋势外推 设线性趋势方程为 $T_t = a + bt$，其中 $t$ 是时间序号。

用最小二乘法拟合趋势线（后续课程详细讲解）。

步骤2：预测公式

加法模型预测：

$$\hat{Y}_{t} = T_t + \bar{S}_{\text{对应季度}}$$

乘法模型预测：

$$\hat{Y}_{t} = T_t \times \bar{S}_{\text{对应季度}}$$

Python完整实现

# ============================================
# 完整时间序列分析流程：从原始数据到预测
# ============================================
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
from sklearn.linear_model import LinearRegression

# 1. 原始季度数据
data = [80, 60, 70, 120, 90, 65, 75, 135, 100, 72, 85, 150]
quarters = [f'{y}Q{q}' for y in range(2021, 2024) for q in range(1, 5)]
df = pd.DataFrame({'Quarter': quarters, 'Y': data})

# 2. 计算4期移动平均（未中心化）
df['MA4'] = df['Y'].rolling(window=4).mean()

# 3. 计算中心化移动平均 = 趋势值 T
df['Trend'] = df['MA4'].rolling(window=2).mean()

# 4. 计算季节指数（乘法模型）
df['Seasonal_Ratio'] = df['Y'] / df['Trend']

# 5. 提取各季度的季节指数并取平均
# 为每个数据点标注季度编号（1-4）
df['Q_Num'] = [1,2,3,4] * 3

# 只取有Trend值的行（去掉头部和尾部的NaN）
valid = df.dropna(subset=['Trend'])

# 计算各季度的平均季节指数
seasonal_index = valid.groupby('Q_Num')['Seasonal_Ratio'].mean()
print("各季度平均季节指数:")
print(seasonal_index.round(3))

# 6. 趋势外推：用时间序号对Trend做线性回归
valid['t'] = range(1, len(valid)+1)
X = valid['t'].values.reshape(-1, 1)
y = valid['Trend'].values
model = LinearRegression()
model.fit(X, y)

print(f"\n趋势方程: T = {model.intercept_:.2f} + {model.coef_[0]:.2f} * t")

# 7. 预测未来4个季度（t=10,11,12,13）
future_t = np.array([[10], [11], [12], [13]])
future_trend = model.predict(future_t)
future_q = [1, 2, 3, 4]

# 8. 叠加季节指数得到预测值
forecast = future_trend * [seasonal_index[q] for q in future_q]

print("\n2024年预测:")
for i, (q, tr, fc) in enumerate(zip(future_q, future_trend, forecast)):
    print(f"  2024Q{q}: 趋势={tr:.1f}, 季节指数={seasonal_index[q]:.3f}, 预测值={fc:.1f}")

# 9. 可视化
plt.figure(figsize=(12, 6))
plt.plot(df['Y'], 'o-', label='实际值', color='#8B6F4E')
plt.plot(df['Trend'], 's--', label='趋势(CMA)', color='#C49A6C')
plt.plot(range(9, 13), forecast, '^--', label='预测值', color='#E74C3C')
plt.legend()
plt.title('时间序列分解与预测')
plt.xticks(range(12), quarters, rotation=45)
plt.tight_layout()
plt.show()

本节小结

  • 季节指数计算：加法 $S = Y - T$，乘法 $S = Y / T$
  • 各年份同一季度的季节指数取平均，得到"典型"季节指数
  • 预测公式：加法 $\hat{Y} = T + \bar{S}$，乘法 $\hat{Y} = T \times \bar{S}$
  • 完整流程：原始数据 → 移动平均 → 中心化 → 提取季节指数 → 趋势外推 → 预测
  • Python可用 rolling().mean() 和 LinearRegression 实现全流程
