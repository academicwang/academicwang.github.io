大数据概念
约 45 分钟

什么是大数据？

大数据（Big Data）是指规模巨大、类型多样、处理速度快到传统数据处理工具难以应对的数据集合。

简单来说，当数据量从"几千条Excel能处理"变成了"几十亿条、实时产生、来自各种渠道"时，我们就进入了大数据的范畴。

大数据的5V特征

业界常用5个以V开头的英文单词来概括大数据的核心特征：

V
Volume（大量）

数据体量巨大，从TB（太字节）到PB（拍字节）级别。全球每天产生约2.5EB数据，相当于全世界所有人每天写满5000万张DVD。

V
Variety（多样）

数据来源和格式极其多样：数据库表格、网页日志、社交媒体帖子、传感器读数、图片、视频、语音等。不再是单一的"结构化表格"。

V
Velocity（高速）

数据产生和处理的速度极快。金融交易系统每秒处理百万笔交易，社交媒体每秒产生数十万条推文，需要实时或准实时分析。

V
Value（价值）

大数据的价值密度很低。海量数据中可能只有1%是有用的信息。就像从一座沙山里淘出一粒金子，需要用强大的算法来提炼价值。

V
Veracity（真实性）

数据质量参差不齐，包含不确定性、噪声和偏差。社交媒体上的虚假信息、传感器故障产生的异常读数、用户填写表单时的随意输入……这些都会影响数据分析结果的可靠性。因此，数据清洗和质量控制是大数据分析的关键步骤。

三种数据类型

根据数据的组织程度，可以分为三大类：

1. 结构化数据（Structured Data）
具有预定义的数据模型，可以用行和列组织。比如关系型数据库中的表格、Excel表格。
示例：客户订单表（订单号、客户ID、金额、日期）
2. 半结构化数据（Semi-structured Data）
没有严格的表结构，但带有标签或标记来组织信息。比如XML、JSON、日志文件。
示例：一条JSON格式的用户行为记录： 
{"user_id": "U001", "action": "click", 
 "item": "product_A", "timestamp": "2024-01-15T09:23:00Z"}
3. 非结构化数据（Unstructured Data）
没有预定义格式的数据，占企业数据的80%以上。比如文本、图片、音频、视频。
示例：客服通话录音、产品评论文字、社交媒体图片、监控摄像头视频

大数据在企业中的主要用途

相关成本原则与限制因素分析
约 60 分钟 · 含完整推导

什么是相关成本？

在企业决策中，管理者最关心的不是"一共花了多少钱"，而是"这个决策会让未来的现金流发生怎样的变化"。这就引入了"相关成本"的概念。

相关成本的三个条件

条件1：未来发生 — 只有未来的成本/收入才与决策相关。过去的支出无法改变。

条件2：因决策而不同 — 在两个备选方案中，如果某个成本无论选哪个方案都一样发生，那它就不相关。

条件3：现金流 — 必须是实际的现金流入或流出，会计上的账面折旧等非现金项目通常不相关。

沉没成本（Sunk Cost）是已经花掉的钱，无论现在做什么决策都无法收回。因此，沉没成本不是相关成本。

经典例子：你花50元买了一张电影票，看了20分钟发现电影很难看。此时50元是沉没成本， relevant的决策依据应该是"继续看是否值得接下来的时间"，而不是"已经花了50元"。

限制因素（Limiting Factor）的概念

企业常常面临一种情况：想生产的产品很多，但资源不够。这个"不够的资源"就是限制因素，也叫瓶颈（Bottleneck）。

常见的限制因素包括：

  • 产能限制：机器运转时间有限，无法同时生产所有产品
  • 材料限制：原材料供应不足或采购预算有限
  • 人工限制： skilled workers 数量不足
  • 资金限制：可用于投资的资金有限

单位限制因素贡献毛益

当资源有限时，企业需要回答一个关键问题：把有限的资源优先用在哪种产品上，才能让总利润最大化？

直觉可能会说"生产单价利润最高的产品"，但这不一定正确！正确的思路是：计算每单位限制资源能带来的贡献毛益。

定义 贡献毛益（Contribution Margin）= 销售价格 − 变动成本
$$CM = P - VC$$
单位限制因素贡献毛益 每种产品每消耗1单位限制资源所带来的贡献毛益：
$$\text{单位限制因素贡献毛益} = \frac{\text{单位贡献毛益}}{\text{单位产品所需限制因素}}$$
决策规则 按照单位限制因素贡献毛益从高到低排序，优先安排排序靠前的产品，直到资源用尽。

最优生产组合决策：完整步骤

步骤1 识别限制因素（瓶颈资源）及其可用总量
步骤2 计算每种产品的单位贡献毛益：$CM = P - VC$
步骤3 计算每种产品单位所需限制因素
步骤4 计算并排序：单位限制因素贡献毛益 = $\frac{CM}{\text{单位限制因素}}$
步骤5 按排序依次分配产能，直到瓶颈资源耗尽
步骤6 计算总贡献毛益 = $\sum$（各产品产量 × 单位贡献毛益）

完整例题：限制因素下的最优生产决策

例题：XYZ公司生产优化

XYZ公司生产A、B、C三种产品，数据如下：

项目产品A产品B产品C
售价（元/件）100150200
变动成本（元/件）6090120
机器工时（小时/件）258
最大需求（件）500300200

公司总机器工时为 3000小时。求最优生产组合和最大总贡献毛益。

解答：

步骤1：限制因素 = 机器工时，总量 = 3000小时

步骤2：计算单位贡献毛益

产品售价变动成本单位贡献毛益 CM
A1006040
B1509060
C20012080

步骤3：计算单位限制因素贡献毛益

产品单位CM机器工时/件单位工时贡献毛益排序
A402$40/2 = \mathbf{20}$1
B605$60/5 = \mathbf{12}$2
C808$80/8 = \mathbf{10}$3

关键发现：虽然产品C的"单位利润"最高（80元），但它消耗工时最多，每小时只贡献10元；而产品A虽然"单位利润"最低（40元），但每小时能贡献20元！所以应该优先生产A。

步骤4：按排序分配产能

顺序产品最大需求所需工时可用工时实际产量
1A500件500×2=1000h3000h500件（全产）
2B300件300×5=1500h3000-1000=2000h300件（全产）
3C200件200×8=1600h2000-1500=500h500/8=62件（剩余产能）

步骤5：计算总贡献毛益

$$\text{总贡献毛益} = 500 \times 40 + 300 \times 60 + 62 \times 80$$ $$= 20{,}000 + 18{,}000 + 4{,}960 = \mathbf{42{,}960 \text{ 元}}$$

结论：最优组合为 A=500件、B=300件、C=62件，最大总贡献毛益为 42,960元。

本节小结

  • 相关成本：未来发生、因决策而不同、涉及现金流。沉没成本不相关。
  • 限制因素 = 瓶颈资源，常见的有产能、材料、人工、资金
  • 单位限制因素贡献毛益 = $\frac{\text{单位贡献毛益}}{\text{单位产品所需限制因素}}$
  • 决策步骤：找瓶颈 → 算单位贡献 → 排序 → 按序分配 → 算总贡献
  • "单位利润最高"不一定最优，关键是"每单位瓶颈资源的贡献"最高

Python实现限制因素优化
约 50 分钟 · 自动化决策与可视化

用Python自动化限制因素分析

当产品种类较多或数据频繁变化时，手工计算容易出错。Python可以帮助我们：

  • 用 pandas DataFrame 统一管理产品数据
  • 自动计算单位贡献毛益和单位限制因素贡献毛益
  • 按排序自动分配产能
  • 一键计算总贡献毛益
  • 用 matplotlib 可视化最优组合

完整Python代码：从数据到最优决策

import pandas as pd
import numpy as np
import matplotlib.pyplot as plt

# ============================================
# 步骤1：定义产品数据
# ============================================
data = {
    '产品': ['A', 'B', 'C'],
    '售价': [100, 150, 200],
    '变动成本': [60, 90, 120],
    '机器工时_per_unit': [2, 5, 8],
    '最大需求': [500, 300, 200]
}

df = pd.DataFrame(data)

# ============================================
# 步骤2：计算单位贡献毛益和单位工时贡献
# ============================================
df['单位贡献毛益'] = df['售价'] - df['变动成本']
df['单位工时贡献'] = df['单位贡献毛益'] / df['机器工时_per_unit']

# 按单位工时贡献排序（降序）
df_sorted = df.sort_values('单位工时贡献', ascending=False)
df_sorted = df_sorted.reset_index(drop=True)

print("=== 各产品单位限制因素贡献毛益 ===")
print(df_sorted[['产品', '单位贡献毛益', '机器工时_per_unit', '单位工时贡献']])

# ============================================
# 步骤3：按排序自动分配产能
# ============================================
total_hours = 3000  # 总机器工时
remaining_hours = total_hours
results = []

for idx, row in df_sorted.iterrows():
    product = row['产品']
    max_demand = row['最大需求']
    hours_per_unit = row['机器工时_per_unit']
    cm_per_unit = row['单位贡献毛益']
    
    # 计算该产品最多能生产多少
    max_possible = min(max_demand, remaining_hours // hours_per_unit)
    used_hours = max_possible * hours_per_unit
    contribution = max_possible * cm_per_unit
    
    results.append({
        '产品': product,
        '排序': idx + 1,
        '计划产量': max_possible,
        '使用工时': used_hours,
        '贡献毛益': contribution
    })
    
    remaining_hours -= used_hours

result_df = pd.DataFrame(results)
print("\n=== 最优生产组合 ===")
print(result_df)

# ============================================
# 步骤4：计算总贡献毛益
# ============================================
total_contribution = result_df['贡献毛益'].sum()
print(f"\n总贡献毛益: {total_contribution:,.0f} 元")
print(f"剩余未用工时: {remaining_hours} 小时")

# ============================================
# 步骤5：可视化最优组合
# ============================================
fig, axes = plt.subplots(1, 2, figsize=(14, 5))

# 柱状图：最优产量组合
colors = ['#C49A6C', '#8B6F4E', '#E8D5C4']
axes[0].bar(result_df['产品'], result_df['计划产量'], color=colors, edgecolor='white')
axes[0].set_title('最优生产组合', fontsize=14)
axes[0].set_ylabel('产量（件）')
for i, v in enumerate(result_df['计划产量']):
    axes[0].text(i, v + 5, str(v), ha='center', fontweight='bold')

# 饼图：各产品贡献占比
axes[1].pie(result_df['贡献毛益'], labels=result_df['产品'], autopct='%1.1f%%',
          colors=colors, startangle=90)
axes[1].set_title('各产品贡献毛益占比', fontsize=14)

plt.tight_layout()
plt.show()

模拟多产品生产环境

在实际业务中，数据可能来自CSV文件或数据库。我们可以构建一个通用的限制因素优化函数：

def optimize_production(df_products, bottleneck_hours, bottleneck_col):
    """
    通用限制因素优化函数
    
    参数:
        df_products: DataFrame，包含产品信息
        bottleneck_hours: 限制因素总可用量
        bottleneck_col: 限制因素对应的列名（如'机器工时_per_unit'）
    
    返回:
        最优生产方案 DataFrame
    """
    # 确保按单位限制因素贡献排序
    df = df_products.copy()
    df['单位限制因素贡献'] = df['单位贡献毛益'] / df[bottleneck_col]
    df = df.sort_values('单位限制因素贡献', ascending=False)
    
    remaining = bottleneck_hours
    plan = []
    
    for _, row in df.iterrows():
        max_make = min(row['最大需求'], remaining // row[bottleneck_col])
        used = max_make * row[bottleneck_col]
        plan.append({
            '产品': row['产品'],
            '产量': max_make,
            '使用资源': used,
            '贡献毛益': max_make * row['单位贡献毛益']
        })
        remaining -= used
    
    return pd.DataFrame(plan)

# 使用示例
products = pd.DataFrame({
    '产品': ['A', 'B', 'C', 'D'],
    '单位贡献毛益': [40, 60, 80, 25],
    '机器工时_per_unit': [2, 5, 8, 1],
    '最大需求': [500, 300, 200, 800]
})

result = optimize_production(products, 3000, '机器工时_per_unit')
print(result)
